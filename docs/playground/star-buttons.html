Star Button Playground

Seven approaches to merging the "View on GitHub" button with a live star count. Pick one by number and I'll swap it into the hero on docs/index.html.

Live count from limehawk/prinstall · fetched on page load via api.github.com

Variant 01 Reference: shadcn/ui, GitHub native

Two-segment flat — the canonical

Star —
Pros

Instantly recognizable — looks like the native GitHub "Star" button. Neutral colors blend with any theme. Works in tight layouts.

Cons

Blends in a little too well — easy to overlook on a busy page. Doesn't use the orange brand accent at all.

Variant 02 Reference: custom, brand-aligned

Two-segment with orange count

Star on GitHub —
Pros

Brand-aligned (orange pops), the count is the visual anchor, shadow on hover gives it weight. High click-through design.

Cons

Two prominent orange elements on the page (this and the Download CTA) can compete for attention.

Variant 03 Reference: Dub.co, Documenso

Rounded pill with bullet separator

View on GitHub · —
Pros

Soft and approachable shape. The bullet separator is classy without being loud. Orange count adds the brand touch without a solid fill.

Cons

Fully rounded shape doesn't match the Download button's 8px radius — might look inconsistent in the CTA row.

Variant 04 Reference: Tailwind CSS docs

Ghost minimal — star + count only

★ —
Pros

Maximum minimalism — assumes the visitor knows what the star means. Tiny footprint in the CTA row. Orange accent on hover.

Cons

No "GitHub" label means first-time visitors might not instantly know what it is. Low visual weight.

Variant 05 Reference: matches current "Download for Windows" button

Solid accent CTA with count pill

Star on GitHub ★ —
Pros

Matches the existing Download button exactly — same padding, radius, orange fill. Count in a pill is the clearest "this is the star count" signal. Strong call to action.

Cons

Makes the hero have TWO solid orange buttons side-by-side, which can dilute which is the primary action. Download should still be king.

Variant 06 Reference: Vercel, Cal.com secondary CTAs

Secondary button with inline count badge

View on GitHub ★ —
Pros

My recommendation. Reads as "View on GitHub" first (familiar action), count is a small badge inside like a notification pill. Preserves the secondary-button hierarchy so Download stays the hero CTA. Same 8px radius as the other buttons.

Cons

Slightly wider than the other buttons in the row because of the inline pill.

Variant 07 Reference: dev-flavor custom, matches prinstall terminal theme

Terminal command-style (dev flavor)

$ gh repo star limehawk/prinstall ✓ Starred · — stargazers
Pros

Terminal-native flavor that matches the whole prinstall aesthetic. Feels like you're reading the output of a command. Devs will love it. Different from every other landing page star button.

Cons

Wider than a normal button so it changes the CTA row layout. Two-line design is more attention-grabbing but takes more vertical space.
